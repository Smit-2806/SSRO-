07 — Rigor

Three validation methods, reported honestly

We deliberately show all three, including the ones that complicate the story. A model that only reports its best number hasn't been stress-tested.

What this actually tells us: under a fair, leakage-controlled split, the model reaches R² ≈ 0.47 — real signal, but a naive persistence baseline (assume tomorrow's AQI equals today's) achieves lower MAE at this data scale. With only 5 cities and one year, that's the expected outcome, not a flaw in the approach: feature importance confirms all three TROPOMI channels (NO2, HCHO, CO) carry genuine predictive signal, and the architecture, pipeline, and validation methodology are proven end-to-end. The stated next step is scaling training data to India's full ~500+ CAAQMS station network — the exact condition our data pipeline (Objective 1 spec) was designed for — not changing the model.

09 — Roadmap

What this is, and what it isn't yet

  • This is a 5-city, 1-year proof-of-concept — not yet the pan-India, multi-year system the ISRO objective calls for.
  • Pollutant sub-values in the CPCB reference data are formula-derived from official AQI, not per-pollutant sensor readings — standard practice for stations not reporting all 6 pollutants, but worth stating plainly.
  • Scaling to India's ~500+ CAAQMS stations is the direct next step to close the gap between the optimistic (0.92) and rigorous (0.47) validation numbers.
  • INSAT-3D AOD fusion (half-hourly diurnal signal) is specified in the architecture but not yet pulled into this training run — MOSDAC access is the next data source to integrate.
  • SHAP-based feature attribution is specified as a secondary explainability cross-check; this build uses input-gradient attribution, which is faster but a coarser signal.
